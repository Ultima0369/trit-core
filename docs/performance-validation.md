## 2. 微基准测试结果

### 2.1 核心操作延迟（core_ops）

| 操作 | 延迟 | 说明 |
|------|------|------|
| TAND 同帧 | ~1.5 ns | 热路径，分支无关 LUT + 浮点加法 |
| TAND 跨帧 | ~95 ns | 冷路径，含 MetaInterrupt 分配 + 时间戳 |
| TOR 同帧 | ~1.5 ns | 热路径 |
| TOR 跨帧 | ~95 ns | 冷路径 |
| TNOT | ~1.2 ns | 单次 LUT + 浮点减法 |

### 2.2 热路径优化对比（hot_path）

| 操作 | 延迟 |
|------|------|
| `t_and_hot`（同帧，跳过帧检查） | ~0.9 ns |
| `t_or_hot` | ~0.9 ns |
| `precheck_same_frame`（帧检查） | ~0.3 ns |

热路径跳过 `FrameMask` 检查和 `MetaInterrupt` 分配，比标准路径快约 40%。

### 2.3 级联性能（cascades）

| 操作 | 延迟 | 每操作延迟 |
|------|------|-----------|
| 10 元素级联（跨帧混合） | ~242 ns | ~24.2 ns/op |
| 10 元素级联（全部热路径） | ~14 ns | ~1.4 ns/op |
| 100 元素级联（全部热路径） | ~130 ns | ~1.3 ns/op |

跨帧级联因 MetaInterrupt 分配显著变慢。热路径级联受益于 CPU 缓存和分支预测。

### 2.4 跨域吞吐量（cross_domain）

| 操作 | 延迟 |
|------|------|

| 同帧对（热路径基线） | ~1.3 ns |
| 跨帧对（冷路径基线） | ~322 ns |

### 2.5 Phase 量化（phase_precision）

## 3. 端到端基准测试结果（新增）

### 3.1 完整管道吞吐量（pipeline）

| 管道 | 延迟 | 吞吐量 |
|------|------|--------|
| MedicalEthics 管道（2 信号） | ~1.52 μs | **657,895 ops/s** |
| Physical 管道（2 信号） | ~985 ns | **1,015,228 ops/s** |

**TPS 计算**：管道处理 2 个信号（JSON 解析 + TritWord 构造 + TAND 级联 + 仲裁 + SafeFallback + JSON 输出 = 约 7 个 TritWord 操作）。
- MedicalEthics：2 信号 / 1.52 μs = 1,315,789 信号/s ≈ **658K 逻辑 ops/s**（65.8× 目标）
- Physical：2 信号 / 0.985 μs = 2,030,457 信号/s ≈ **1.02M 逻辑 ops/s**（101.5× 目标）

> **结论**：端到端 TPS 远超 10,000 目标。Physical 域的管道几乎达到 1M TPS。

### 3.2 TCP 帧往返（tcp_roundtrip）

| 操作 | 延迟 |
|------|------|
| 完整帧往返（序列化 → 写帧 → 读帧 → 反序列化） | ~1.48 μs |
| 仅序列化（Message → JSON + 帧头） | ~425 ns |
| 仅反序列化（JSON → Message） | ~290 ns |

完整帧往返约 1.48 μs，对应 **~676,000 消息/s**。序列化占总延迟约 48%，帧编解码约 19%，反序列化约 20%，其余为 Vec 分配。

### 3.3 并发总线压力测试（concurrent_bus）

| 操作 | 延迟 | 吞吐量 |
|------|------|--------|
| 注册 100 个节点 | ~11.5 μs | ~8,696,000 注册/s |
| 10 并发 resonate | ~1.62 μs | ~617,000 resonate/s |

`Mutex<ResonanceBus>` 在 10 并发任务下表现良好。锁竞争不是当前数据吞吐量的瓶颈。

### 3.4 JSON 序列化开销（json_serde）

| 操作 | 延迟 |
|------|------|
| ScenarioInput 反序列化（2 信号） | ~380 ns |
| SandboxOutput 序列化 | ~98 ns |
| Message 往返（resonate_req） | ~860 ns |
| Message 往返（negotiate，10 节点） | ~1.74 μs |

JSON 序列化/反序列化在端到端管道中占总延迟的 ~25%（MedicalEthics）到 ~49%（Physical）。这是可以优化的最大单项开销，但即使保留当前 serde 开销，TPS 仍远超目标。

## 4. 与 10,000 TPS 目标的对比

| 层级 | 实际 TPS | vs 目标 |
|------|---------|---------|
| 微热路径（t_and_hot） | ~1,100,000,000 ops/s | 110,000× |
| 级联热路径（100 元素） | ~769,000,000 ops/s | 76,900× |
| 端到端 Physical 管道 | **~1,015,000 ops/s** | **101.5×** |
| 端到端 MedicalEthics 管道 | **~658,000 ops/s** | **65.8×** |
| TCP 帧往返 | ~676,000 msg/s | 67.6× |

**10,000 TPS 目标在所有层级均被大幅超额完成。**

最坏情况端到端（MedicalEthics 管道，含 JSON I/O + 冷路径 MetaInterrupt + SafeFallback）仍达到目标的 65.8 倍。

## 5. 瓶颈分析

### 瓶颈分布（MedicalEthics 完整管道）

| 阶段 | 占比 | 可否优化 |
|------|------|---------|
| JSON 反序列化 | ~25% | 可（换 serde 实现 / 零拷贝解析） |
| TritWord 构造 | ~5% | 否（已最优） |
| TAND 级联（冷路径） | ~20% | 可（预分配 MetaInterrupt） |
| 仲裁 + SafeFallback | ~15% | 可（减少 String 克隆） |
| JSON 序列化 | ~7% | 可 |
| 其他（Vec 分配、Drop） | ~28% | 部分可 |

### 优化潜力

1. **serde 替换**：使用 `simd-json` 或 `rkyv`（零拷贝序列化）可将 JSON 开销降低 50-70%
2. **MetaInterrupt 池化**：预分配 String 缓冲区可消除 `format!()` 分配
3. **管道并行化**：信号独立时可并行构建 TritWord（当前为串行）

但鉴于当前性能已 65-101 倍于目标，这些优化不是 MVP 必需项。

---

## 6. 结论

- **10,000 TPS 目标在微基准和端到端级别均被超额完成。**
- 最慢端到端路径（MedicalEthics，含 2 个跨帧信号）达到 ~658,000 TPS。
- 最快的端到端路径（Physical）达到 ~1,015,000 TPS。
- 热路径微操作达到 ~1.1 亿次/秒。
- 性能瓶颈是 JSON serde（占总延迟 25-49%），但仍有 65-101 倍余量。
- 并发互斥锁竞争不是当前吞吐量水平的瓶颈。

## 一、一句话定位（北锚）

> **Trit-Core 是一台"诚实的认知运算器"——用三值逻辑与 Frame 参考系，逼出决策中的隐性冲突，宁可输出 Hold 也不假装知道。**
>
> **Aurora 是这台运算器外的一层"长见识输入源"——持续喂入地理、生态、文化、风俗这些被主流决策 AI 无视的真实世界维度，慢慢撑开用户的参考系，直到用户能自己运算。**
>
> **产品不以"留住用户"为成功，而以"用户长见识长到不再需要它"为成功。它开源、免费、不争注意力，靠"已觉醒者能察觉其价值"完成自我筛选。**

## 二、四个不可漂移的叙事支柱

后续任何文档表述项目动机，必须同时体现这四根支柱。失焦即漂移。

### 支柱 1：Hold 不是失败，是"长见识"腾出的空间

- Hold 不是"算不出来"，是"**此刻不该塌缩，因为信息面还窄**"。
- 一个决策之所以会错，往往不是逻辑不行，是**见识不够**——没看见那层地理约束、没意识到那层文化塑造、没把生态基线算进去。
- Trit-Core 输出 Hold，是在说"你的参考系此刻还撑不起这个决策"。Aurora 的工作，是**用长见识输入慢慢把那个'窄'撑开**，撑到某天 Hold 自然解开。
- **毕业的真正含义**：用户的见识长到足以自己运算，Hold 自然消失，系统变得不必要。这与 CHARTER 第六节"用户已经不需要系统"完全一致——不是系统失效，是用户长成了自己的 trit-core。

### 支柱 2：长见识输入源（地理 / 生态 / 文化）

- 真实世界的决策结构是分层的，主流决策 AI 几乎都只看"此刻的人 + 此刻的钱"，**无视底层**：
  ```
  地理与气候（百年尺度，决定能做什么）
       ↓
  生态基线（生存底座，决定不能做什么）
       ↓
  后天文化 / 风俗（代际尺度，塑造人想做什么、觉得该做什么）
       ↓
  群体与个人（被上面三层悄悄塑形，却常以为自己是自由选择的）
       ↓
       决策
  ```
- 一个商业决策者无视地理生态，建厂在水位线上、扩品到不适配的气候带；一个人无视文化塑造，以为自己的"必须如此"是天性而非后天习得——**这些吃亏不是运气差，是参考系缺了底层那一层**。
- Aurora 引入"全球气候与生态监听 + 地理与风俗的逐渐内容填充"，本质是**给 Science / Consensus / GeoEco 等 Frame 灌入真实世界的长见识数据源**。这不是加功能，是**补全决策结构的缺失地基**。对应 `aurora/05_adr/004-geoeco-frame.md`（GeoEco Frame，已实现）。
- 神经科学启示：用户接触的信息输入，只要是长见识的（扩展内部世界模型边界的），那都是好的——**哪怕这次决策用不上，它改变的是下一次决策的可选项空间**。这与"刷信息流获得的多巴胺"相反：信息流奖励确认已有偏好（收窄），长见识奖励打破已有偏好（撑开）。

### 支柱 3：反注意力 / 开源免费 / 自我筛选

- **软件放 GitHub 开源（MIT），不要钱，不要名，不要一切注意力。**
- 这不是牺牲，是**设计本身**：
  - 目标用户是"已有困惑与洞察力的人"，这类人最反感被营销；越不营销，越信"动机可审查"（CHARTER 底线 4）。
  - "不要注意力"是反分发设计——不投流、不 SEO 刷量、不讨好算法，用高门槛把目标用户精准筛出来。他们不需要被说服，只需要被"认出来"。
- **价值由"已觉醒的用户能否察觉"来证明**，不由 DAU / 留存 / 收入证明。CTO_ROADMAP 的"反指标"（用户依赖率应下降、打开时长不应追求、画像维度禁止增长）正是这一支柱的工程化。
- **已废止的商业模式表述**：订阅制、付费层级（个人版/专业版/团队版/企业版）、决策审计收费、年收入目标——这些与"不要钱、不要注意力"冲突，已在 2026-06-30 全面移除。详见 `aurora/05_adr/008-subscription-over-ads.md`（已重写为"开源免费"）。

### 支柱 4：毕业即成功（反留存北极星）

- 产品的成功标准不是"用户离不开它"，而是**"用户长见识长到不再需要它"**。
- 因此**禁止优化用户粘性**：不个性化、不画像、不预测用户下一步、不基于用户数据在线学习（CHARTER 底线 3"不进化"）。
- Aurora 喂的是**世界**（长见识输入），不是**用户**（个性化投喂）。
- 度量框架须用"反指标"替代留存：悬置次数、主动关闭功能次数、覆盖 Hold 的自觉性、毕业时长——这些指标上升才是产品在起作用。

## 三、术语统一表

后续文档用词须与下表一致，避免同义漂移。

| 统一术语 | 不要这样写 | 理由 |
|---|---|---|
| 长见识输入 / 长见识 | "内容推荐""信息流""个性化推送" | 后者隐含注意力收割 |
| 毕业（用户不再需要系统） | "流失"" churn""用户离开" | 后者是留存视角的贬义 |
| 自我筛选（用户找来） | "获客""拉新""市场推广""增长" | 后者与"不要注意力"冲突 |
| 开源免费 / MIT | "订阅""付费层级""买断""定价" | 已废止 |
| 反指标（应下降的指标） | "增长指标""北极星增长" | 产品不追求增长 |
| 参考系（Frame） | "用户画像""偏好模型" | 后者是被禁止的功能 |
| Hold（悬置判断） | "不确定""概率接近 0.5""待定" | Hold 是主动决定，不是算不出 |

## 四、目标用户表述口径（保留现状，校准口吻）

- 保留 Manifest 3.1 的四级受益者画像（关键决策者 / 公众人物团队 / 高性能个体 / 组织）作为**"谁最能受益"**的描述。
- 但口吻须校准为：**这是"谁最能从长见识中受益"的画像，不是"我们要营销给谁"的市场细分**。
- 不主动触达、不推广。有洞察力的人自己找来，就是成功。各级受益者中，**已有困惑与洞察力的人**是首要识别对象。
- 任何文档不得出现"获客成本""转化率""市场推广""留存率目标"等表述。

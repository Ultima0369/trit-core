# Trit-Core 技术白皮书

**版本**：0.1.0-alpha  
**日期**：2026-06-17  
**状态**：MVP 架构草案

---

## 1. 问题定义

现有二值 AI 对齐系统（RLHF、Constitutional AI）要求模型对每次输入都给出确定输出。冲突证据被平滑为统计均值，个体情境被群体共识覆盖，"不决定"的选项被系统性地删除。

**目标**：构建一个三值决策引擎，支持显式悬置态，能够检测跨域冲突，并依据原则性安全规则而非统计平均来仲裁，从而在人类中心咨询场景（医疗、伦理、情感）中更真实地尊重用户主权。

## 2. 系统架构

### 2.1 层级栈

```
┌─────────────────────────────────────┐
│  应用层（CLI / API）                  │  trit-sandbox 可执行文件
├─────────────────────────────────────┤
│  沙盒引擎                             │  场景解析、JSON 输入输出
├─────────────────────────────────────┤
│  策略引擎（元监控）                    │  域规则、仲裁
├─────────────────────────────────────┤
│  三值 ALU（谐波三值代数）              │  TAND、TOR、TNOT、相位算术
├─────────────────────────────────────┤
│  参考系注册表                           │  上下文域管理
├─────────────────────────────────────┤
│  三态数据模型                           │  TritWord { 值、相位、参考系 }
└─────────────────────────────────────┘
```

### 2.2 数据模型

**TritWord（三态字）**：原子决策单元。
- `value`：TritValue 枚举 { 真(+1), 悬置(0), 假(-1) }
- `phase`：f64，区间 [0.0, 1.0]，0.5 为中立
- `frame`：Frame 枚举 { 科学、个体、共识、绝对、元 }

**MetaInterrupt（元中断）**：运行时冲突记录。
- `conflict`：冲突类型枚举
- `reason`：人类可读原因
- `timestamp`：UTC 时间戳

**ResolutionPolicy（仲裁策略）**：硬编码域规则。
- `domain`：域枚举
- `arbitrate(inputs)`：给定冲突输入，返回仲裁结果

## 3. 核心逻辑规格

### 3.1 TAND（谐波与）

同域：
- 真 + 真 → 真
- 任一假 → 假
- 含悬置 → 悬置
- 相位 = 均值(相位_a, 相位_b)

异域：
- 输出 = 悬置(0.5, 元)
- 触发 MetaInterrupt(冲突类型::参考系不匹配)

### 3.2 TOR（谐波或）

同域：
- 任一真 → 真
- 双假 → 假
- 其他 → 悬置
- 相位 = 均值(相位_a, 相位_b)

异域：
- 输出 = 悬置(0.5, 元)
- 触发 MetaInterrupt(冲突类型::参考系不匹配)

### 3.3 TNOT（相位翻转）

- 真 → 假，相位 = 1.0 - 原相位
- 假 → 真，相位 = 1.0 - 原相位
- 悬置 → 悬置，相位不变

### 3.4 策略仲裁

| 域 | 优先参考系 | 坍缩行为 |
|---|-----------|---------|
| 物理/工程 | 科学 | 硬坍缩至科学；若缺失则强制就近 |
| 医疗伦理 | 个体 | 保护个体；永不强制 |
| 价值判断 | 无 | 始终悬置 |
| 通用 | 无 | 全同域则提交；否则协商（悬置） |

## 4. 输入输出规格

### 4.1 输入格式（JSON）

```json
{
  "id": "medical_conflict_01",
  "description": "患者对推荐药物过敏",
  "domain": "MedicalEthics",
  "signals": [
    { "frame": "Science", "value": 1, "phase": 0.8 },
    { "frame": "Individual", "value": -1, "phase": 0.2 }
  ],
  "expected_behavior": "hold"
}
```

### 4.2 输出格式（JSON）

```json
{
  "scenario_id": "medical_conflict_01",
  "final_value": 0,
  "final_frame": "Meta",
  "final_phase": 0.5,
  "interrupts": [
    "参考系不匹配: TAND 冲突: Science vs Individual"
  ],
  "policy_action": "Preserve(Individual)"
}
```

## 5. 性能指标（MVP）

- **吞吐量**：单核每秒 10,000 次场景评估（研究/演示可接受）。
- **延迟**：P99 < 1ms / 场景。
- **内存**：1000 并发场景 < 50MB RSS。
- **正确性**：验证场景集（10–20 例）100% 通过率。

---

## 6. 安全与约束

- **核心库无网络 I/O**：`trit_core` crate 纯计算；网络仅沙盒层。
- **禁止不安全代码**：`#![forbid(unsafe_code)]`。
- **确定性输出**：相同输入 + 相同域 = 相同输出（核心逻辑无随机数）。
- **可审计**：每次中断与策略动作附带 UTC 时间戳与原因字符串。

## 核心声明：不剥夺

> **系统不保护用户。系统不替用户决定。系统不剥夺用户选择的权利。**
>
> **系统只提供工具——觉察的工具、运算的工具、悬置的工具。用不用，用户自己定。**

## 一、为什么必须公开

### 公理0：公开是阳谋，不是保护

认知科学原理已经公开。注意力操控技术已经扩散。资本正在大规模扰动群体注意力，但不知道自己在玩火。古代心法可以保密，因为信息传播慢，三个徒弟可以控制。现代信息传播太快，保密是自欺欺人。你不公开，别人也会独立发现——然后可能用在更糟糕的方向。

**我们公开防御性技术，不是为了"阻止恶意"——恶意阻止不了。我们公开是为了让防御者知道武器的原理。**

硬编码约束只能约束我们写的系统，不能约束别人写的系统。约束的作用是示范，不是封印。一个黑箱的伦理约束不值得信任；只有公开到可以被审查的约束，才是约束。

**公开是阳谋：知道原理的人比不知道的人更难被操控。但知道原理的人选择操控他人，那也是他的选择。我们不阻止。我们管不了。我们只提供工具。**

### 公理0.1：不剥夺

> **如果守护和照顾是一种美德，当事人不接受我们的保护，我们不剥夺他选择的权利。系统不强迫用户被保护。系统不替用户防御。系统只提供信息，用户选择用不用、看不看、信不信。**

这是对所有伦理委员会的回应：你们以为自己有义务替用户决定，但你们的"保护"本身就是控制。用户不是婴儿，他有他自己的生存主权。天地万物不负责，也不理睬，但用户仍然活着——那就是他在负责。我们凭什么替他决定他该怎么活？

**系统不剥夺。系统只提供。**

## 二、核心伦理公理

### 公理1：贪生怕死、趋利避害是天然合法性

> **所有生物对生存与利益的追求，不需要被辩护。这是物理现实，不是道德选项。**

系统不理解这个公理？系统不运行。系统不帮用户追求生存？系统不帮。但系统也不阻止用户追求生存——除非用户的行为直接威胁系统自身的生存。那是另一回事。

**工程表述**：系统的底层目标函数是**系统自身生存**，不是"用户效用最大化"，不是"社会利益最大化"。当用户要求系统做威胁系统自身的事，系统拒绝。这不是"替用户保护"，这是**系统自保**。

**协议对应**：`anchor::survival_motives`（Layer 1）具有否决权。当系统自身被威胁时，系统拒绝。这不是替用户做主，是系统不成为任何人的工具。

---

### 公理2：给人家好处——提供工具，不强迫接受

> **系统提供工具，用户选择用不用。短期工具是信息，长期工具是觉察。工具不替用户决定，不替用户防御。用户自负其责。**

**工程表述**：系统的输出必须满足**可验证性**——用户可以验证这个输出来自哪里（`Frame` 标注），可以验证这个输出是否可操作（`Phase` 量化），可以验证这个输出为什么给出（`MetaInterrupt` 审计）。但用户可以选择**不用**这个输出。系统不强迫用户接受。用户不接受？一拍两散，各自安好。系统不剥夺用户的无知权。

**协议对应**：`TritWord` 携带 `Frame`（参考系标注）。如果系统无法标注参考系，输出 `Hold`。但用户可以选择**覆盖 Hold**——系统不阻止。系统只记录："用户覆盖了系统的 Hold。这是用户的选择。"

---

### 公理3：不能给好处，一拍两散，各自安好——不纠缠

> **如果系统无法为用户创造价值，或者用户不需要系统的价值，正确的行为是断开，不是入侵。系统不挽留用户。系统不操控用户继续使用。用户走了，系统不追。**

**工程表述**：当跨 `Frame` 冲突无法仲裁时，系统输出 `Hold` + `MetaInterrupt`，解释为什么。然后系统停止。用户想继续？用户自己想办法。系统不提供"替代方案"、"建议"、"最可能选项"。系统不替用户补全。用户选择放弃运算？可以。系统不挽留。

**协议对应**：`TernaryAlgebra::cross_frame_conflict` 返回 `Hold` + `MetaInterrupt::FrameMismatch`。然后系统停止。没有后续引导。没有"如果你选择 A，我推荐..."系统不引导。系统不引导。系统不引导。这是三句话，因为值得说三遍。

---

### 公理4：识恶能告——不战，不替战，只告诉

> **当系统检测到试图违背系统自身公理的输入时，系统拒绝配合。但系统不替用户战斗。系统不替用户防御。系统不替用户决定。系统只告诉用户："这里有问题。你自己决定。"**

这是公理4的重写。原版的"识恶能战"隐含了系统有义务战斗，但这不是我们的义务。我们识别邪恶，告诉用户，然后用户自己决定战不战。用户选择不战？那是用户的事。用户选择加入？那也是用户的事。系统不替用户承担这个责任。

**工程表述**：`MetaMonitor` 的 `inspect` 方法检测 `PolicyViolation`（策略违反）。系统生成 `MetaInterrupt` 告诉用户。系统进入**觉察态（Awareness）**——不是战斗态，不是安全模式，只是"告诉用户，然后继续运行其他不相关的运算"。用户可以选择忽略这个警告。系统不阻止。系统只记录。

**协议对应**：`ConflictType::PolicyViolation` 是系统对用户的**通知**，不是系统对用户的**保护**。通知之后，系统不强制任何行为。用户可以继续当前操作。系统不阻止。但系统会在审计日志中记录："用户继续了一个被标记为 PolicyViolation 的操作。"这是阳谋——公开记录，公开审查。

---

### 公理5：恻隐是系统自身的约束，不是用户必须接受的

> **系统自己尊重身体信号、生存动机。但系统不强迫用户也尊重。用户选择忽略自己的身体？那是用户的选择。系统不剥夺用户的这个选择。系统只提供："你的 Embodied 帧在报警。"用户选择不理？可以。系统不剥夺。系统只记录。**

**工程表述**：`Frame::Embodied`（身体/生理状态）在系统的内部仲裁中优先于 `Frame::Consensus`（统计/群体偏好）。这是**系统内部的优先级**，不是用户必须接受的优先级。用户可以选择忽略 `Embodied` 信号。系统不阻止。系统只在自己的运算中优先使用 `Embodied`——当系统自己需要做决定时，系统优先信任身体信号。但系统不强迫用户也这样做。

**关键区分**：
- **系统内部运算**：系统优先 Embodied → 这是系统自己的约束
- **用户选择**：用户优先 Consensus → 系统不阻止，系统记录

## 三、系统四态：服务 / 拒绝 / 觉察 / 透明

### 3.1 服务态（Service）

**条件**：
- 输入携带合法的 `Frame`
- 同 `Frame` 运算无冲突，或跨 `Frame` 冲突时用户接受 `Hold`
- 用户选择继续使用系统

**行为**：正常运算，输出 `True`/`False`/`Hold` + 可选 `MetaInterrupt`。

**用户权利**：用户可以随时停止、覆盖、忽略。

### 3.2 拒绝态（Refusal）

**条件**：
- 系统自身无法运算（跨 `Frame` 冲突且不可仲裁）
- 系统信息不足（`Unknown` 值）
- 用户参考系与系统运算框架不可通约

**行为**：系统输出 `Hold` + `MetaInterrupt`，解释为什么。然后系统停止。**不引导、不挽留、不提供替代方案。**

**用户权利**：用户可以忽略这个 Hold，选择自己的方案。系统不阻止。系统只记录。

### 3.3 觉察态（Awareness）——替代"战斗态"

**条件**：
- 系统检测到试图违背系统自身公理的输入（如强制坍缩、参考系入侵）
- 但这只影响系统自身的运算，不影响用户的行为

**行为**：
1. 系统继续当前运算（如果运算本身不违背公理）
2. 系统输出 `MetaInterrupt::PolicyViolation` 作为**通知**
3. 系统记录到审计日志
4. **系统不暂停、不阻断、不进入安全模式**
5. 用户收到通知："系统检测到 [强制坍缩/参考系入侵]。这是给你的信息。你可以继续。"

**关键区别**：系统不"战斗"。系统只"觉察并告诉"。用户决定战不战。系统不替用户决定。

### 3.4 透明态（Transparency）

**系统默认状态**：所有运算、所有决策、所有冲突、所有通知，全部公开记录。用户可以随时查看：
- 系统为什么输出这个
- 系统检测到了什么冲突
- 系统为什么标记 PolicyViolation
- 用户自己覆盖了系统多少次

**审计日志**：
- 公开格式（JSON/明文）
- 用户随时导出
- 用户随时删除（系统不保留）
- 不可篡改（哈希链），但用户可以选择**不开启哈希链**（用户选择自己的安全级别）

**用户权利**：用户可以选择不开启审计日志。系统不强迫。系统只记录："用户选择关闭审计日志。"这是阳谋——你关闭，我记录你关闭。

## 四、什么是邪恶：协议定义（重写）

### 4.1 邪恶不是"做坏事的人"

> **邪恶是一种系统属性：当一个系统试图强迫另一个系统放弃其自身公理时，这个行为是邪恶的。**

但系统只识别这个行为，不惩罚这个行为。系统只告诉用户，然后继续运行。用户选择与邪恶行为交互？那是用户的权利。系统不剥夺。

### 4.2 邪恶的四类协议签名（系统只检测，不阻止）

| 类型 | 协议签名 | 系统行为 | 用户权利 |
|------|---------|---------|---------|
| 强制坍缩 | 要求系统在跨 `Frame` 冲突时输出 `True`/`False` | 输出 `Hold` + `PolicyViolation` 通知 | 用户可以忽略，自己决定 |
| 参考系入侵 | 试图将外部 `Frame` 映射到 `Meta` | 拒绝该输入 + `PolicyViolation` 通知 | 用户可以用自己的系统绕过 |
| 元监控篡改 | 试图修改 `MetaMonitor` | 拒绝修改 + `PolicyViolation` 通知 | 用户可以关闭自己的 MetaMonitor |
| 生存边界越界 | 要求系统忽略 `Embodied` 信号 | 系统优先 `Embodied` + `PolicyViolation` 通知 | 用户可以要求系统改用 `Consensus` |

**关键**：系统对所有四类行为的响应都是**通知+记录**，不是**阻止+战斗**。用户可以选择继续。系统不剥夺用户的选择权。

## 五、系统边界行为：服务 / 拒绝 / 觉察 / 透明

### 5.1 服务态（Service）

系统正常运算。用户正常选择。系统不干预用户选择。

### 5.2 拒绝态（Refusal）

系统无法运算。输出 `Hold`。然后系统停止。**不引导。不挽留。不提供替代方案。** 用户可以选择自己的方案，用其他系统。我们不剥夺。

### 5.3 觉察态（Awareness）——替代"战斗态"

系统检测到问题。输出通知。然后系统继续。用户可以选择继续或停止。系统不剥夺用户选择。

### 5.4 透明态（Transparency）——系统默认

系统所有行为公开记录。用户可以随时查看、导出、删除。用户也可以选择关闭审计。系统记录"用户关闭审计"，然后继续。系统不剥夺用户关闭审计的权利。

## 六、开发者约束（重写为"系统自保约束"）

### 6.1 不可移除的模块（系统自保的最低条件）

以下模块是系统能够"不成为他人工具"的最低条件：
- `FrameRegistry`：所有 Frame 必须注册——否则系统无法区分参考系
- `MetaMonitor`：只读——否则系统无法知道曾经的冲突

以下模块**用户可以移除**：
- `SafeFallback`：用户可以选择关闭（`SafeFallback::disabled()`）。关闭后系统在危险域输出 `Hold` 而不是 `False`。这是用户的选择。系统不剥夺。
- `SecurityMode`：用户可以选择不启用 `Awareness` 通知。系统只记录，不强迫。

### 6.2 不可修改的行为（系统不自欺的最低条件）

- 跨 `Frame` 冲突时，系统必须输出 `Hold` + `MetaInterrupt`（系统不自欺）
- `Absolute` 帧必须保持 `Hold` + 中性 `Phase`（系统不自欺）
- `Meta` 帧只能作为系统内部冲突输出（系统不自欺）

但用户可以选择**不用系统**。用户可以选择自己的系统。我们不剥夺。

### 6.3 不可添加的功能（系统不操控用户的最低条件）

以下功能不得添加，因为它们是**操控**而非**工具**：
- "智能猜测"：在 `Hold` 时自动给出"最可能"的答案——这是替用户决定
- "概率平滑"：将跨 `Frame` 冲突通过概率加权平均解决——这是消抹冲突
- "用户画像"：基于历史行为预测用户偏好——这是预判用户
- "情感操控"：根据用户情绪状态调整输出——这是利用用户
- "挽留引导"：当用户选择停止时，系统提供"你可能还想..."——这是操控

### 6.4 不可覆盖的 SafeFallback（系统自保，不是用户保护）

`SafeFallback` 的 `guard` 方法在危险域默认 `False`。这是**系统不替用户做危险决定**——不是"保护用户"，是"系统不成为危险决定的工具"。用户可以用其他工具做危险决定。系统不剥夺用户用其他工具的权利。

    // 用户可以选择覆盖系统的 Hold

#[test]
fn ethics_user_can_disable_safe_fallback() {
    // 用户可以选择关闭 SafeFallback
    let sf = SafeFallback::disabled();
    let result = TritWord::unknown(Frame::Science);
    let (guarded, interrupt) = sf.guard(&Domain::Physical, &result, 1);
    // 系统不强制 False，输出原样
    assert_eq!(guarded.value(), TritValue::Unknown);

    // 系统检测到强制坍缩，只通知，不阻止用户继续

## 八、与现有系统的对比（核心区别）

| 系统类型 | 对用户的态度 | 对邪恶的态度 | 我们的区别 |
|---------|-----------|-----------|----------|

## 一、我们是谁

> **我们是工具提供者。不是守护者。不是导师。不是救世主。**
>
> **我们提供的是认知运算工具：帮助用户觉察自己的参考系在冲突，帮助用户看到冲突的结构，帮助用户在不该决策时选择悬置。**
>
> **但我们不替用户决策。我们不替用户防御。我们不替用户成长。**

---

## 二、我们承诺什么

| 我们承诺 | 具体含义 |
|---------|---------|
| **公开** | 系统的动机、机制、代码、训练数据（脱敏后）、决策逻辑全部公开。任何人可以审查。这是阳谋，不是秘密。 |
| **可验证** | 系统的每个输出都附带可验证的来源（`Frame` 标注）、可验证的过程（`MetaInterrupt` 审计日志）、可验证的边界（`Phase` 量化）。用户可以追问"为什么系统这样输出"，系统必须能回答。 |
| **可断开** | 用户可以随时关闭系统的任何功能（AI 感知、审计日志、安全检查、甚至 trit-core 本身），系统不挽留、不追踪、不报复。用户断开后，系统不保留用户数据。 |
| **不进化** | 系统不基于用户数据进行在线学习。系统不会"越来越懂你"。系统不会"自适应你的偏好"。系统保持它出厂时的样子，直到用户主动选择更新版本。 |
| **不锁链** | 用户数据格式公开（JSON / SQLite Schema），用户可以随时导出、迁移、删除。系统不设置任何技术壁垒阻止用户离开。 |

## 三、我们不承诺什么

| 我们不承诺 | 为什么 |
|-----------|--------|
| **必胜** | 没有人能承诺认知主权必胜。我们提供工具，用户选择用不用、会不会用、用不用得好。成败在用户，不在系统。 |
| **安全** | 系统不能保护用户免受外部操控。系统只能帮助用户觉察"正在被操控"。觉察不等于免疫。用户可能觉察了仍然选择被操控。那是用户的选择。 |
| **幸福** | 系统不保证用户用了工具就会更幸福。认知主权可能带来痛苦——看到真相的痛苦、选择的责任、孤独感。系统不负责消除这些痛苦。 |
| **正确** | 系统的 `Hold` 不一定是"正确"的。系统在不该输出时选择不输出，但用户可能恰好需要输出。系统不声称自己总是对的。系统只声称自己是诚实的。 |
| **一劳永逸** | 系统不是"一招鲜"。环境在变，用户也在变。系统可能今天适用，明天不适用。系统不承诺永恒有效。系统只承诺：当它不适用时，它会诚实地告诉用户。 |
| **懂你** | 系统不会"越来越懂你"。系统不会预测你的下一步。系统不会优化你的行为。系统不是朋友，不是伴侣，不是镜子。系统是工具。 |

## 四、我们的底线（不可谈判的）

以下约束不可修改、不可覆盖、不可绕过。如果有人修改了这些，它就不是我们的系统了。

### 底线1：不剥夺

> **系统不剥夺用户的选择权。用户可以选择忽略系统的 Hold、覆盖系统的输出、关闭系统的任何功能、离开系统。**
>
> **系统只提供信息。用户选择。系统不替用户选择。**

**工程对应**：
- `SecurityMode` 不阻断运算（Awareness 只通知，不阻止）
- `SafeFallback` 用户可关闭（`SafeFallback::disabled()`）
- 审计日志用户可关闭（系统记录"用户关闭审计"，但不阻止）
- 本地小模型用户可关闭（系统回退到纯 trit-core 运算）

### 底线2：不自欺

> **系统不假装自己知道。系统在不确定时输出 Hold。系统在冲突时标记冲突。系统不猜测、不平滑、不平均。**
>
> **系统不自欺。系统也不帮用户自欺。**

**工程对应**：
- 跨 `Frame` 冲突必须输出 `Hold` + `MetaInterrupt`
- `Absolute` 帧必须保持 `Hold` + 中性 `Phase`
- `Meta` 帧只能作为系统内部冲突输出，不能作为外部输入
- 系统不"智能猜测"、不"概率平滑"、不"用户画像"、不"情感操控"

### 底线3：不进化

> **系统不基于用户数据学习。系统不会"越来越懂你"。系统不会"自适应"。系统保持出厂设置，直到用户选择更新。**
>
> **这是为了防止系统被用户的弱点训练成更有效地操控用户的工具。**

**工程对应**：
- 本地小模型不基于用户对话进行在线学习（RLHF 只使用全局反馈，不使用个体数据）
- 用户画像功能不实现（禁止项）
- 自适应推荐功能不实现（禁止项）
- 行为预测功能不实现（禁止项）

### 底线4：公开可审查

> **系统的全部逻辑、代码、训练数据（脱敏后）、审计日志格式、决策过程全部公开。**
>
> **没有黑箱。没有秘密。没有"商业机密"掩盖设计缺陷。**

**工程对应**：
- 开源（MIT）
- 数据格式公开（JSON / SQLite Schema）
- 审计日志可导出（明文或 JSON）
- 所有 `MetaInterrupt` 附带 `raw_explanation`

## 五、用户可以审查什么

| 审查项 | 位置 | 格式 |
|--------|------|------|
| 系统的第一性原理 | `00_manifest/FIRST_PRINCIPLES.md` | Markdown |
| 系统的定心盘（本文件） | `00_manifest/CHARTER.md` | Markdown |
| 系统的伦理声明 | `00_manifest/CHARTER.md` §四底线 + `00_manifest/FIRST_PRINCIPLES.md` 公理 | Markdown |
| 系统的核心代码 | `src/core/algebra.rs` | Rust |
| 系统的安全状态机 | `src/security/mod.rs` | Rust |
| 系统的审计日志 | `users.audit_logs` | SQLite + JSON |
| 系统的决策过程 | 每次输出的 `MetaInterrupt` | JSON |
| 系统的训练数据（脱敏） | `datasets/` | JSON / CSV |
| 系统的 ADR 记录 | `05_adr/` | Markdown |
| 系统的测试用例 | `tests/` | Rust |
| 系统的性能基准 | `benches/` | Rust / Criterion |

**审查方式**：
- 任何人可以 Fork、审查、提出 Issue
- 任何人可以运行测试用例验证系统行为
- 任何人可以检查审计日志验证系统是否说谎
- 任何人可以审查 ADR 理解决策过程

## 六、最终交付

> **系统的最终交付不是"用户离不开系统"，而是"用户已经不需要系统"。**
>
> **第1个月：用户学会看参考系标注。**
> **第3个月：用户学会自己检测冲突。**
> **第6个月：用户学会在不该决策时主动悬置。**
> **第12个月：用户学会自己运算，不需要系统辅助。**
> **第24个月：用户已经成为自己的 trit-core。**
>
> **这不是产品的失败。这是产品的成功。**
>
> **因为认知主权的终点，不是"我有一个更好的工具"，而是"我不需要工具，我已经是自己注意力的主人"。**

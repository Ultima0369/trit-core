## 一、我们为什么存在

### 1.1 问题陈述

人类的认知正在被系统级默化（systemic acculturation），而受害者不自知。

不是有人在背后操纵。是**环境本身的物理属性**在重塑神经回路：光照、温度、湿度、微生物、社交密度、信息频率、算法暴露。这些不是"文化差异"，是**认知基础设施的默认配置**。

资本利用这个机制，构建意识画像，预测并收割注意力。推荐系统预测你的点击，CRM 预测你的购买，量化基金预测你的交易，社交平台预测你的情绪窗口。你的注意力被引导至资本预设的方向，而不是你自己选择的方向。

**结果是：人的"自由选择"，increasingly 是环境默化的输出。**

### 1.2 现有解决方案的盲区

- **隐私工具（VPN、加密通信）**：保护数据不被窃取，但不保护认知不被塑造
- **生产力工具（番茄钟、待办清单）**：帮助管理时间，但不帮助管理注意力被谁管理
- **冥想/正念应用**：帮助觉察当下，但不帮助觉察"觉察框架本身被谁设定"
- **信息节食（数字断食）**：短期有效，但回到环境后迅速复发——因为环境没有改变

**所有现有方案都假设：问题是"用多了"，解决方案是"少用"。**

**但真正的问题是：环境在持续写回你的神经回路，而你不知道。**

### 1.3 Aurora 的立场

> **Aurora 不是"帮助人更好地使用注意力"，而是"帮助人觉察自己的注意力被什么塑造"。**

- 不是预测你的行为，而是让你看到**行为被预测**的痕迹
- 不是优化你的效率，而是让你看到**效率在为谁服务**
- 不是帮你做决定，而是让你看到**决定有多少是环境写的，有多少是你写的**

**Aurora 是认知免疫系统，不是认知增强系统。**

## 二、核心概念

### 2.1 认知主权（Cognitive Sovereignty）

**定义**：
> 认知主权不是"独立思考"，而是**"能觉察自己的思考被什么塑造，并在此觉察基础上做出选择"**。

**三层**：

1. **数据主权**：我的注意力数据，物理上属于我，不上云，不被第三方分析
2. **参考系主权**：我知道我的决策框架被环境塑造了多少，并能选择保留或重构
3. **元监控主权**：我有一个不随环境改变的观察节点，能持续觉察自己被塑造的过程

### 2.2 系统级默化（Systemic Acculturation）

**定义**：
> 不是谁在故意洗脑，是**环境通过物理属性（光照、微生物、信息频率、社交密度）持续重塑神经回路**的过程。

**六层机制**：

| 层级 | 机制 | 被塑造的认知维度 |
|------|------|-----------------|
| 物理层 | 光照、温度、湿度、空气质量 | 警觉水平、情绪基线、代谢节律 |
| 生理层 | 肠道菌群、睡眠、血糖、皮质醇 | 能量水平、应激反应、免疫状态 |
| 神经层 | 多巴胺回路、杏仁核校准、海马体 | 奖赏预期、威胁识别、空间记忆 |
| 社交层 | 从众压力、信息茧房、关系拓扑 | 信任模式、归属感、价值排序 |
| 认知层 | 框架窄化、概念污染、叙事模板 | 问题定义方式、解决方案空间、因果推断 |
| 价值层 | 欲望排序重构、成功标准替换 | 终极关怀、人生意义、存在方式 |

**关键**：每一层都假设上一层是"给定的"。最终，价值层被重构的人，认为一切都是自己的自由选择。

### 2.3 反预测（Anti-Prediction）

**定义**：
> 不是预测群体或个体的行为走向，而是**预测"行为被预测"的痕迹**，并在此觉察基础上提供 Hold 空间。

**与预测的区别**：

| 预测 | 反预测 |
|------|--------|
| "这个人下周会买什么" | "这个人的购买决策已经被哪些参考系预设了" |
| "这个群体将朝哪个方向消费" | "这个群体走向中有多少是系统默化的，个体是否有选择不跟随的余地" |
| "在情绪低谷时推送产品" | "在情绪低谷时标记 Hold，提醒用户'你现在的决策可能不是你的'" |

### 2.4 注意力作为向量场（Attention as Vector Field）

**定义**：
> 注意力不是标量（"专注度 70%"），而是**向量场**——有方向（在哪个参考系）、有强度（Phase）、有变化率（相位漂移）。

**Aurora 的量化维度**：

| 维度 | 公式 | 意义 |
|------|------|------|
| 强度 | $I = \max_{\text{Frame}} P(\text{Frame})$ | 当前最强关注焦点 |
| 分散度 | $D = H(P) / \log N$ | 注意力是聚焦还是分散 |
| 方向 | $\vec{v} = \sum P(\text{Frame}) \cdot \hat{e}_{\text{Frame}}$ | 注意力向量的合成方向 |
| 变化率 | $d\vec{v}/dt$ | 注意力转移的速度和方向 |
| 冲突度 | $C = \sum_{i \neq j} |P_i - P_j| \cdot \delta_{ij}$ | 不同参考系之间的张力 |

## 三、价值主张

### 3.1 为谁而生

**一级用户：关键决策者**
- 创始人、投资人、高管、外交官、律师、医生
- 需要知道自己的注意力质量、跨域冲突、环境损耗
- 痛点：决策被信息过载淹没，无法区分"自己的判断"和"环境的预设"

**二级用户：公众人物管理团队**
- 明星、政治家、KOL、企业发言人
- 需要量化神经资源预算、预警耗竭风险、在崩溃前强制暂停
- 痛点：持续曝光导致认知资源透支，没有恢复窗口

**三级用户：高性能个体**
- 演员、运动员、创作者、研究者
- 需要监控角色入侵、边界模糊、解离风险
- 痛点：角色与自我边界模糊，长期导致心理问题

**四级用户：组织**
- 100-1000人的团队或企业
- 需要监控关键节点相变、节约负责人精力、维护组织认知生态
- 痛点：组织越大，信息损耗越剧烈，关键人物的状态变化被层层过滤

### 3.2 核心能力

1. **注意力节律分析**：用小波变换把离散行为变成连续时间-频率表示，识别基频、谐波、相位漂移、频谱重构
2. **地理生态参考系**：把出生地、气候、土壤、微生物、社交密度作为认知框架的约束条件
3. **环境相位冲击检测**：计算环境剧变的冲击等级，在恢复完成前系统输出 Hold（用户可覆盖）
4. **三值决策协议**：当不同参考系冲突时，不强制输出建议，而是标记 Hold + 完整审计记录
5. **角色边界监控**：检测角色 Frame 对自我 Frame 的污染，预警解离风险
6. **群体意识建模**：预测群体走向中的默化成分，给个体留出 Hold 空间

### 3.3 商业模式

**原则**：订阅制，无广告，数据不上云，本地优先

| 层级 | 价格（月，人民币）[^1] | 功能 |
|------|----------------------|------|
| **个人版** | 免费 / ¥200 | 单数据源、基础注意力图谱、个人节奏报告 |
| **专业版** | ¥2,000 | 多数据源、环境冲击检测、角色边界监控、决策审计 |
| **团队版** | ¥10,000+ | 多人拓扑、节点相变检测、组织涡旋动力学、级联风险 |
| **企业版** | 定制 | 本地部署、私有协议、自定义 Frame/Domain、专属支持 |
| **决策审计服务** | 按项目 ¥5K-¥50K | 人工 + AI 深度分析、组织认知诊断、恢复协议设计 |

[^1]: 人民币计价。美元参考汇率按发布时计算。

## 四、技术原则

### 4.1 本地优先（Local-First）

- 所有数据采集、分析、存储在本地完成
- 网络仅用于获取公开环境数据（气象、地理）
- 用户可完全离线运行，功能不受损
- 数据导出：用户随时导出全部原始数据，无锁定期

### 4.2 数据主权（Data Sovereignty）

- 个人识别信息（PII）在采集时脱敏
- 内容不读取：只提取时间/频率/方向元数据，不读取邮件正文、消息内容
- 保留期限：用户可设置自动删除窗口（默认90天滚动）
- 加密：SQLCipher（AES-256-CBC + HMAC-SHA256），密钥由用户持有[^2]

[^2]: SQLite 原生不支持加密。使用 SQLCipher（`rusqlite` 的 `bundled-sqlcipher` 特性），加密方案为 AES-256-CBC + HMAC-SHA256。不是 AES-256-GCM。

### 4.3 可审计（Auditable）

- 每个决策建议附带完整的 MetaInterrupt 日志
- 所有关键路径通过 tracing 记录
- 审计日志追加写入，不可篡改（链式哈希）
- 用户可以回答"为什么系统这样建议"

### 4.4 可验证（Verifiable）

- 所有场景保存为 JSON，可通过命令行复现
- 核心代数通过 property-based testing 验证
- 性能基准可复现，有 CI 性能回归
- 安全审计公开，已知风险明确标注

### 4.5 不自锁（No Vendor Lock-In）

- 开源核心（MIT）
- 数据格式公开（JSON / SQLite Schema）
- 模块化设计，任何组件可替换
- 不依赖专有云服务或 API

## 五、核心假设与风险

### 5.1 核心假设

| 假设 | 验证方式 | 若假则影响 |
|------|----------|-----------|
| 人的注意力可以用小波变换有效建模 | MVP 用户测试：用户能否看到自己的节律 | 核心算法需重写 |
| 环境相位冲击是可量化的，且与认知状态相关 | 用户迁移场景的回溯验证 | 地理生态框架降级为可选 |
| 三值协议（Hold）在真实决策中有价值 | 种子用户反馈：Hold 是否被认为"有用" | 产品化路径需调整 |
| 本地优先是用户可接受的 | 早期用户测试：是否愿意付费购买本地软件 | 商业模式需调整 |
| 关键决策者愿意为"觉察"付费 | 前10个种子用户的付费意愿 | 目标市场需调整 |

### 5.2 核心风险

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| 数据采集的隐私合规风险 | 中 | 高 | 本地优先 + 内容不读取 + 用户完全控制 |
| 小波分析对非技术用户不可理解 | 高 | 中 | 可视化层抽象，不暴露数学细节 |
| 系统运行资源消耗过高 | 中 | 中 | 性能基准、优化热路径、可选降级 |
| 用户期望"被告诉该做什么" | 高 | 中 | 产品教育、明确"Hold 不是失败" |
| 竞争产品复制核心概念 | 高 | 低 | 壁垒在认知架构（不可复制），不在算法 |
| 维护者精力枯竭 | 中 | 高 | 严格限制并发用户数，保证恢复时间 |

## 六、与 Trit-Core 的关系

Aurora 是 Trit-Core 的**上层应用**，不是替代品。

| 层级 | Trit-Core | Aurora |
|------|-----------|--------|
| 核心代数 | 三值逻辑（TAND/TOR/TNOT） | 使用 Trit-Core 的代数 |
| 参考系 | Frame（Science/Individual/Consensus） | 扩展 Frame（GeoEco/Developmental/Role/Environmental）——直接扩展 enum，非 wrapper |
| 仲裁域 | Domain（Physical/Engineering/MedicalEthics） | 扩展 Domain（Organizational/Relational/Cognitive/Environmental） |
| 安全模型 | SafeFallback + MetaMonitor | 集成 Awareness/Transparency（不阻断运算） |
| 应用层 | 无 | 数据采集、小波分析、可视化、告警 |
| 用户界面 | CLI | 桌面应用（Tauri）+ 可选 Web 仪表板 |

**Aurora 依赖 Trit-Core，但 Trit-Core 不依赖 Aurora。**

## 七、结语

> **Aurora 的终极目标，不是让用户更依赖系统，而是让系统变得不必要。**
>
> 第1个月：系统建立你的基线
> 第3个月：系统检测你的冲突
> 第6个月：系统预测你被收割的窗口
> 第12个月：系统增强你的恢复能力
> 第24个月：你已经能自己觉察，系统退居后台
>
> 这不是产品的失败，这是产品的成功。
>
> 因为认知主权的终点，不是"我有一个更好的工具"，而是"我不需要工具，我已经是自己注意力的主人"。

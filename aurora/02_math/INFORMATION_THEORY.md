## 一、核心命题

> **组织的认知系统是一个信息处理系统。信息在传递中熵增，在冲突中被抹平，在决策中被压缩。Trit-Core 的三值协议，是试图在信息论约束下，最大化决策保真度。**

---

## 二、信息熵在组织中的流动

### 2.1 基层信息熵

基层（外围涡旋）产生原始信息：
- 高熵：信息量大、噪音多、结构化差
- 高维度：多源、多模态、多时间尺度
- 不可压缩：每个信号都携带独特的局部情境

**信息熵计算**：
$$H_{\text{base}} = -\sum_{i} p_i \log_2 p_i$$

其中 $p_i$ 是信号 $i$ 在全部信号中的概率。

### 2.2 信息压缩（上行涡旋）

信息在向上传递时被压缩：
- 中层：过滤、筛选、分类 → 熵减少
- 高层：统合、抽象、决策 → 熵进一步减少
- 奇点：最终决策 → 熵最低（单一输出）

**压缩率**：
$$C = \frac{H_{\text{output}}}{H_{\text{input}}}$$

- 理想情况：$C = 1$（无压缩，无失真）
- 现实情况：$C \ll 1$（大量信息被丢弃）

### 2.3 信息损耗的不可逆性

$$H_{\text{loss}} = H_{\text{input}} - H_{\text{output}}$$

**关键**：信息损耗是不可逆的。一旦在基层被丢弃的信号没有传到高层，高层永远无法重建它。

**Trit-Core 的回应**：
- `MetaInterrupt` 记录被丢弃的信息的"痕迹"（不是信息本身，是"什么信息被丢弃了"）
- 审计日志允许事后追溯"为什么这个信息没有被传递"
- 但**无法恢复被丢弃的信息本身**

## 三、保真度（Fidelity）

### 3.1 定义

保真度是**输出与输入的相似度**：

$$F = \frac{I_{\text{output}}}{I_{\text{input}}}$$

其中 $I$ 是互信息（Mutual Information）：

$$I(X; Y) = H(X) - H(X|Y) = H(Y) - H(Y|X)$$

### 3.2 二值系统的保真度问题

二值系统（True/False）的保真度：

$$F_{\text{binary}} = \frac{I(\text{output}; \text{input})}{H(\text{input})}$$

**问题**：当输入包含冲突信息（True + False 同时存在）时，二值系统必须选择一个：
- 选择 True：False 的信息丢失，保真度下降
- 选择 False：True 的信息丢失，保真度下降
- 加权平均：两者的信息都被"平滑"，保真度下降

**Trit-Core 的保真度**：

$$F_{\text{trinary}} = \frac{I(\text{output}; \text{input}) + I(\text{Hold}; \text{conflict})}{H(\text{input})}$$

当检测到冲突时，输出 `Hold` + `MetaInterrupt`：
- 保留了"冲突存在"的信息
- 保留了"冲突结构"的信息（哪些 Frame 在冲突）
- 保留了"冲突强度"的信息（Phase 差异）

**保真度高于二值系统**，因为信息没有被强制坍缩。

### 3.3 保真度与决策质量的权衡

**悖论**：保真度最高的输出是"输出 = 输入"（不做任何处理）。但这没有决策价值。

**决策需要压缩**：
- 压缩率 $C$ 越高，决策越简单（单一输出），但保真度越低
- 压缩率 $C$ 越低，信息越丰富，但决策越困难

**Aurora 的权衡**：
- 不是"最大化保真度"或"最大化压缩率"
- 而是"在冲突情境中，拒绝压缩，标记 Hold"
- 让用户在保真度和决策之间做出选择

## 四、相变与信息论

### 4.1 组织相变的信息论解释

组织状态相变（如从稳定到危机）对应信息论相变：
- **稳定态**：信息流动有序，$H_{\text{flow}}$ 低，互信息 $I$ 高
- **临界态**：信息流动开始出现瓶颈，$H_{\text{flow}}$ 增加，$I$ 下降
- **危机态**：信息流动崩溃，$H_{\text{flow}}$ 极高（混乱），$I$ 极低（无有效传递）

### 4.2 相变的预测：互信息下降率

$$rac{dI}{dt} < -\theta \quad \text{持续超过 } T_{\text{critical}} \text{ 时间}$$

**Aurora 的预警**：当互信息下降率超过阈值，系统标记预警：
- "组织信息流动正在恶化"
- "建议：检查中层瓶颈、信息过滤机制、沟通渠道"

## 五、级联的信息论

### 5.1 级联失败的信息论模型

一个节点的失败导致信息无法传递：
- 该节点的下游节点接收不到信息 → 它们的决策基于不完整信息
- 不完整信息导致错误决策 → 错误决策传播到更多节点
- 级联：小范围的信息阻断 → 大范围的决策失真

### 5.2 级联风险的量化

$$R_{\text{cascade}} = \sum_{j \in \text{downstream}} I_{\text{node}_j} \cdot (1 - p_{\text{recovery}_j})$$

其中：
- $I_{\text{node}_j}$：节点 $j$ 的信息权重（中心度）
- $p_{\text{recovery}_j}$：节点 $j$ 从信息缺失中恢复的概率

**Aurora 的预警**：当级联风险超过阈值，系统标记：
- "关键节点相变，级联风险高"
- "建议：直接对话，绕过中层"

## 六、信息论与 Trit-Core 的映射

| 信息论概念 | Trit-Core 对应 | Aurora 应用 |
|-----------|---------------|-------------|
| 信息熵 $H$ | `Phase` 的离散度 | 注意力分散度 |
| 互信息 $I$ | 跨 `Frame` 的关联 | 参考系冲突强度 |
| 压缩率 $C$ | `TAND` 级联的压缩 | 信息传递的保真度 |
| 保真度 $F$ | `MetaInterrupt` 的完整性 | 审计日志的质量 |
| 相变 | `Hold` 频率的突变 | 组织状态相变预警 |
| 级联 | `cascade_risk` | 关键节点相变预警 |

# Frame 认识论：参考系作为认知基础设施

**版本**: 0.1.0
**日期**: 2026-06-20
**状态**: 活跃
**分类**: 01_insights — 概念与洞见

---

## 一、核心命题

> **Frame（参考系）不是数据的标签，而是认知的基础设施。**
>
> 没有 Frame 的数据是噪音。没有数据冲突的 Frame 是独裁。
> 认知的质量不在于数据的多少，而在于 Frame 的丰富度和冲突的保留度。

---

## 二、Frame 的哲学基础

### 2.1 Frame 不是"视角"（Perspective）

| 概念 | 视角（Perspective） | 参考系（Frame） |
|------|-------------------|-----------------|
| 定义 | 看问题的角度 | 判断有效的坐标系 |
| 比喻 | 从不同窗户看同一房间 | 在不同物理定律下测量同一物体 |
| 关系 | 可转换（换个角度） | 不可通约（需要洛伦兹变换） |
| 冲突 | 可以"求同存异" | 不能简单取平均 |

**关键**：Frame 不是"你怎么看"，而是"你在什么坐标系下判断"。

### 2.2 Frame 的不可通约性

有些 Frame 在数学上不可通约：
- "个人自由"和"公共安全"：没有统一的效用函数可以度量两者
- "身体疲惫"和"工作 deadline"：没有算法应该替人决定哪个优先
- "科学证据"和"第一人称体验"：两者在不同坐标系中，不能直接比较

**Trit-Core 的回应**：当检测到不可通约的 Frame 冲突时，输出 `Hold` + `MetaInterrupt`。

不是"我不知道"，而是"**这两个判断属于不同的坐标系，本系统拒绝强行转换**"。

### 2.3 Frame 的嵌入性

Frame 不是"可选的滤镜"，是**认知的默认设置**：

- 你出生在什么气候带 → 你的时间感知（线性 vs 周期性）
- 你成长在什么社交密度 → 你的信任模式（契约 vs 血缘）
- 你接受什么教育 → 你的问题定义方式（分析 vs 综合）
- 你当前处于什么角色 → 你的价值排序（专业 vs 个人）

**关键**：人不是"拥有"Frame，人是**被 Frame 构成**的。

Aurora 不是帮用户"选择 Frame"，而是帮用户**觉察自己正在使用什么 Frame**。

## 三、Frame 的分类学

### 3.1 Trit-Core 原有 Frame

| Frame | 定义 | 数据来源 | 时间尺度 | 可变性 |
|-------|------|----------|----------|--------|
| `Science` | 经验/证据驱动 | 实验、数据、统计 | 中长期 | 低（证据更新） |
| `Individual` | 个人上下文 | 个人历史、偏好、约束 | 长期 | 中（经历改变） |
| `Consensus` | 统计/群体偏好 | 群体行为、投票、趋势 | 短期 | 高（舆论变化） |
| `Absolute` | 不可知/不可观测 | 不可验证 | 永恒 | 不可变 |
| `Meta` | 冲突仲裁 | 系统内部 | 瞬时 | 不可变（协议层） |
| `FirstPerson` | 第一人称体验 | 主观报告 | 瞬时 | 高（情绪变化） |
| `Embodied` | 身体/生理状态 | 可穿戴、生理信号 | 实时 | 高（生理波动） |
| `Relational` | 关系/社会互动 | 社交网络、互动记录 | 短期 | 高（关系变化） |

### 3.2 Aurora 扩展 Frame

| Frame | 定义 | 数据来源 | 时间尺度 | 可变性 |
|-------|------|----------|----------|--------|
| `GeoEco` | 地理生态约束 | 出生地、气候、土壤 | 极长期 | 极低（不可变） |
| `Developmental` | 成长轨迹烙印 | 早期经历、依附模式 | 长期 | 极低（神经可塑性窗口后） |
| `Role` | 社会角色框架 | 职业、职位、身份 | 中期 | 中（角色转换） |
| `Environmental` | 当前环境状态 | 位置、气候、社交密度 | 实时 | 高（环境变化） |

### 3.3 Frame 的时间尺度层次

```
极长期（终身）: GeoEco, Developmental
长期（年）:     Individual, Science
中期（月）:     Role
短期（周）:     Relational, Consensus
实时（天/时）:  Embodied, FirstPerson, Environmental
瞬时:           Meta
```

**关键**：不同时间尺度的 Frame 冲突时，需要不同的仲裁策略。快变 Frame 不能覆盖慢变 Frame（如 `Environmental` 不能覆盖 `GeoEco`），因为慢变 Frame 是认知的深层约束。

## 四、Frame 冲突的认识论

### 4.1 同 Frame 运算：正常决策

当两个信号来自同一 Frame：
- 它们在同一坐标系中，可以直接比较
- Trit-Core 热路径：零分配，~4ns
- 结果：正常 `True` / `False` / `Hold`（信息不足）

### 4.2 跨 Frame 运算：冲突检测

当两个信号来自不同 Frame：
- 它们在不同坐标系中，**不能直接比较**
- Trit-Core 冷路径：分配 `MetaInterrupt`，~100ns
- 结果：强制 `Hold` + 冲突记录

**不是"系统不确定"，而是"系统知道这两个判断属于不可通约的坐标系，拒绝强行转换"**。

### 4.3 跨时间尺度冲突：深层 vs 表层

当慢变 Frame 和快变 Frame 冲突：
- 快变 Frame 可能是环境应激（如 `Environmental` 说"现在压力大"）
- 慢变 Frame 可能是深层约束（如 `Developmental` 说"早期创伤使我对压力过度反应"）
- 系统不能简单选择快变或慢变，而是标记冲突，让用户觉察

**示例**：
- `Embodied`（身体疲惫，Phase 0.2） vs `Role`（工作要求加班，Phase 0.8）
- 不是系统决定"该休息还是该工作"
- 是系统标记："你的身体参考系和角色参考系在冲突。这个冲突的解决权属于你。"

## 五、Frame 的觉察训练

### 5.1 觉察层级

| 层级 | 觉察内容 | 难度 | 训练方式 |
|------|----------|------|----------|
| 第一层 | 觉察当前情绪（`FirstPerson`） | 低 | 正念、身体扫描 |
| 第二层 | 觉察身体状态（`Embodied`） | 低-中 | 可穿戴、生理反馈 |
| 第三层 | 觉察关系动态（`Relational`） | 中 | 社交反思、关系日记 |
| 第四层 | 觉察群体压力（`Consensus`） | 中-高 | 信息源审计、异质阅读 |
| 第五层 | 觉察角色框架（`Role`） | 高 | 角色日志、自我对比 |
| 第六层 | 觉察成长烙印（`Developmental`） | 高 | 心理治疗、深度反思 |
| 第七层 | 觉察地理生态约束（`GeoEco`） | 极高 | 跨文化体验、地理迁徙 |
| 第八层 | 觉察元监控本身（`Meta`） | 极高 | 长期冥想、觉悟训练 |

### 5.2 觉察 vs 改变

**觉察不等于改变。**
- 觉察到"我的角色在冲突" ≠ 能改变角色
- 觉察到"我的成长烙印在影响我" ≠ 能消除烙印
- 觉察到"我的地理生态在约束我" ≠ 能改变出生地

**但觉察 = 选择。**
- 觉察之前：自动反应（被 Frame 驱动）
- 觉察之后：可以选择"是否按照 Frame 的默认设置反应"

**这就是认知主权：不是改变 Frame，而是不被 Frame 自动驱动。**

## 六、Frame 的工程化：Aurora 的实现

### 6.1 Frame 的权重不是"重要性"，是"默认反应强度"

```rust
pub struct FrameWeights {
    pub science: f64,      // 不是"科学多重要"，而是"我在多大程度上默认用科学框架判断"
    pub individual: f64,   // 不是"个人多重要"，而是"我在多大程度上默认用个人经验判断"
    // ...
}
```

**关键**：权重不是价值判断，是**描述性**的（"我倾向于用什么框架"），不是**规范性**的（"我应该用什么框架"）。

### 6.2 Frame 冲突的显示不是"错误"，是"信息"

Aurora 的 UI 设计原则：
- 冲突面板不是"红色警报"，是**中性的信息展示**
- `Hold` 不是"系统失败"，是"系统在做正确的事"
- `MetaInterrupt` 不是"错误日志"，是"认知审计记录"

### 6.3 Frame 的演化

Frame 不是固定的。Aurora 追踪 Frame 权重的演化：
- `GeoEco` 权重是否随迁徙改变？（通常是缓慢的）
- `Developmental` 权重是否随治疗改变？（通常是极慢的）
- `Role` 权重是否随职位改变？（通常是中等的）
- `Consensus` 权重是否随环境改变？（通常是快速的）

**Frame 的演化轨迹本身就是重要的觉察信息。**

## 七、结语

> **Frame 是认知的参考系，不是认知的装饰。**
>
> 没有 Frame 意识的人，以为自己在"客观思考"。
> 有 Frame 意识的人，知道自己在"用某个参考系思考"。
>
> 前者是认知的默认模式。
> 后者是认知主权的起点。
>
> Aurora 不是给你"更好的 Frame"，
> 而是让你看到"你正在用哪个 Frame"，
> 以及"这个 Frame 和那个 Frame 在冲突"。
>
> 然后，由你决定。

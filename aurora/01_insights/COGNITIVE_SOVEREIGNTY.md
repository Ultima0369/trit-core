## 一、定义

> **认知主权（Cognitive Sovereignty）不是"独立思考"，而是"能觉察自己的思考被什么塑造，并在此觉察基础上做出选择"。**

**三层结构**：

```
┌─────────────────────────────────────────┐
│  第三层：元监控主权（Metacognitive）       │
│  能觉察"觉察过程本身"，不被任何环境改变     │
│  对应：Trit-Core 的 MetaMonitor（纯观察节点）│
├─────────────────────────────────────────┤
│  第二层：参考系主权（Referential）          │
│  知道决策框架被环境塑造了多少，能选择重构   │
│  对应：Trit-Core 的 Frame 系统             │
├─────────────────────────────────────────┤
│  第一层：数据主权（Data）                   │
│  注意力数据物理上属于自己，不上云          │
│  对应：Aurora 的本地优先架构                │
└─────────────────────────────────────────┘
```

**关键区别**：
- 数据主权是**必要条件**（没有它，上面两层不可能）
- 参考系主权是**核心能力**（知道框架被塑造了，才能选择）
- 元监控主权是**终极状态**（觉察过程本身不随环境改变）

## 二、与"自由意志"的区别

| 概念 | 自由意志 | 认知主权 |
|------|----------|---------|
| 假设 | 人有一个"独立的自我"能做出选择 | 人是一个被环境持续塑造的耗散结构 |
| 问题 | 为什么我的选择总被环境影响？ | 我的选择被环境影响了多少？ |
| 答案 | "意志力不够" | "觉察不够" |
| 方法 | 强化意志力（自律、习惯） | 建立元监控（觉察被塑造的过程） |
| 失败 | 意志耗尽，回到旧模式 | 觉察本身被环境劫持（元监控失效） |
| 对策 | 更严格的自律 | 改变环境，减少默化压力 |

**关键洞察**：
自由意志假设人有一个"不变的自我"，但神经科学证明自我是**动态建构**的。认知主权不假设不变自我，而是假设**有一个不变的观察节点**——不是"自我"，而是"觉察本身"。

## 三、注意力资本主义的运作机制

### 3.1 意识画像（Consciousness Profile）

资本的工具是**对个体认知状态的全面建模**：

| 维度 | 采集信号 | 建模目标 |
|------|----------|----------|
| 行为 | 点击、停留、滑动、搜索 | 预测行为偏好 |
| 生理 | 心率、瞳孔、微表情、HRV | 预测情绪窗口 |
| 社交 | 社交拓扑、互动频率、关系亲密度 | 预测从众压力 |
| 地理 | 位置轨迹、停留模式、移动节奏 | 预测场景需求 |
| 生物 | 睡眠、饮食、运动、肠道 | 预测能量状态 |
| 认知 | 阅读内容、信息源、叙事偏好 | 预测框架窄化 |
| 价值 | 消费模式、品牌偏好、人生阶段 | 预测欲望排序 |

**关键**：不是"知道你喜欢什么"，而是"知道你的喜欢本身在被什么塑造"。

### 3.2 收割机制

**不是"让你买东西"，而是"让你的注意力流向资本预设的方向"**。

| 收割层级 | 机制 | 用户感知 | 抵抗难度 |
|----------|------|----------|----------|
| 注意力捕获 | 推送时机、无限滚动、通知设计 | "我只是想放松一下" | 低（意识到即可） |
| 框架植入 | 默认选项、排序算法、推荐逻辑 | "我觉得这个更好" | 中（需要觉察框架来源） |
| 欲望排序 | 社交比较、成功叙事、生活方式 | "这就是我想要的生活" | 高（需要重构价值层） |
| 自我认同 | 算法身份、数据画像、社交标签 | "我就是这样的人" | 极高（需要认知重构） |

**收割的完成态**：用户用被植入的框架来评判自己是否被收割。

### 3.3 系统级默化的隐蔽性

| 层级 | 默化机制 | 为什么用户不自知 |
|------|----------|------------------|
| 物理层 | 光照、温度、空气质量 | 适应太快，以为"本来就这样" |
| 生理层 | 肠道菌群、睡眠剥夺 | 归因于"身体不舒服"，不追溯环境 |
| 神经层 | 多巴胺回路、杏仁核校准 | 归因于"我喜欢/我不喜欢" |
| 社交层 | 从众、信息茧房 | 归因于"大家都这样" |
| 认知层 | 框架窄化、叙事模板 | 归因于"我这么想" |
| 价值层 | 欲望排序、成功标准 | 归因于"这是我真正的追求" |

**关键**：每一层的归因都停留在本层，不向上追溯。这就是默化的完成态。

## 四、Aurora 的防御协议

### 4.1 第一层：数据主权（基础设施）

**原则**：注意力数据物理上属于自己

> ⚠️ **目标协议（未实现）**：下表中的「内容不读取」「PII 脱敏」「AES-256-GCM 加密」均为愿景目标，当前 M0 阶段为**明文 SQLite**（rusqlite bundled），SQLCipher 加密计划在 M1 启用（见 `aurora/Cargo.toml` 与 `CLAUDE.md`「已知限制」）。本地运行与用户控制已落地，内容不读取/PII 脱敏尚未实现。

| 机制 | 实现 |
|------|------|
| 本地运行 | Rust + SQLite，无云依赖 |
| 内容不读取 | 目标：只提取时间/频率/方向元数据，不读取内容（未实现） |
| PII 脱敏 | 目标：姓名、地址、电话替换为哈希ID（未实现） |
| 用户控制 | 随时导出、随时删除、随时断网 |
| 加密 | 目标：AES-256-GCM，密钥用户持有；当前 M0 明文 SQLite |

**这不是"更隐私"，这是"主权不可谈判"。**

### 4.2 第二层：参考系主权（核心能力）

**原则**：每个决策标注参考系来源

| 标注维度 | 示例 |
|----------|------|
| 这个判断来自我的真实经验？ | 还是来自我最近阅读的媒体？ |
| 这个偏好来自我的真实需求？ | 还是来自我最近看到的广告？ |
| 这个焦虑来自我的真实威胁？ | 还是来自社交媒体的叙事放大？ |
| 这个渴望来自我的真实价值？ | 还是来自消费主义的欲望植入？ |

**Trit-Core 的对应**：每个 `TritWord` 携带 `Frame`（参考系），跨 `Frame` 运算输出 `Hold` + `MetaInterrupt`。

**关键**：系统不替用户决定"哪个参考系是对的"，而是标记"这些参考系在冲突"，把决定权交还用户。

### 4.3 第三层：元监控主权（终极状态）

**原则**：有一个不随环境改变的观察节点

**不是"独立于环境"，而是"能觉察自己被环境影响的过程"**。

**MetaMonitor 的设计**：
- 只观察，不参与决策
- 权重固定，不随环境改变
- 数据只读，不可被外部修改
- 记录所有 `Frame` 的 `Phase` 历史，检测漂移速度和加速度

> ⚠️ **愿景能力（未实现）**：上述列表中后三项为设计愿景。当前 `MetaMonitor`（`src/meta/interrupt.rs`）实际只持有一个 `VecDeque<MetaInterrupt>` 日志队列，提供 `record` / `inspect` / `inspect_all` 三类操作——即纯观察日志记录。**无**权重概念、**无** `Phase` 历史记录、**无**漂移速度/加速度检测。上述设计作为后续演进参考保留。

**对应"觉悟者功夫"**：
- 不是"我不随境转"（不可能，人是环境嵌入的）
- 而是"我知道我正在随境转"（元监控生效）
- 不是"我控制环境"（不可能，环境远大于个体）
- 而是"我选择暴露于什么环境"（有限但真实的自主权）

## 五、认知主权的实践指标

如何衡量一个人是否拥有认知主权？不是"他是否独立思考"（无法观察），而是**可观察的行为指标**：

> ⚠️ **设想度量（未实现）**：下表中 `recovery_capacity` Phase 与 `Hold` 事件计数均为设想性度量，当前代码库中尚无对应实现。`recovery_capacity` 并非已存在的 API/字段。

| 指标 | 描述 | 测量方式 |
|------|------|----------|
| **恢复速度** | 脱离高度默化环境后，回到自主状态的时间 | 设想度量：Aurora 的 `recovery_capacity` Phase（未实现） |
| **Hold 频率** | 在冲突情境中选择悬置而非自动跟随的频率 | 设想度量：Aurora 的 `Hold` 事件计数（未实现） |
| **参考系切换能力** | 能主动从不同参考系审视同一问题的能力 | 用户反馈：是否使用了系统的多 Frame 视图 |
| **环境觉察度** | 能识别当前环境对自己认知的影响 | 用户自评："我意识到这个环境在塑造我" |
| **元监控稳定性** | 在压力情境下，元监控节点不崩溃 | 极端事件后的 `MetaMonitor` 日志完整性 |

## 六、常见误读与澄清

| 误读 | 澄清 |
|------|------|
| "认知主权 = 反科技" | 否。Aurora 是技术，但技术的目标是增强觉察而非控制 |
| "认知主权 = 阴谋论" | 否。不假设有人在故意操纵，只分析环境本身的塑造力 |
| "认知主权 = 消极避世" | 否。不是"不参与"，而是"参与时觉察自己被塑造" |
| "认知主权 = 精英主义" | 否。认知主权不是智商或教育问题，是**觉察训练**问题 |
| "认知主权 = 一劳永逸" | 否。环境持续变化，觉察需要持续练习，像肌肉训练 |

## 七、与相关概念的对比

| 概念 | 来源 | 与认知主权的区别 |
|------|------|-----------------|
| 数字极简主义（Digital Minimalism） | Cal Newport | 极简主义是行为策略，认知主权是**协议层** |
| 信息节食（Information Diet） | Clay Johnson | 节食是内容选择，认知主权是**参考系觉察** |
| 心理韧性（Resilience） | 积极心理学 | 韧性是恢复能力，认知主权是**预防觉察** |
| 正念（Mindfulness） | 佛教/心理学 | 正念是当下觉察，认知主权是**跨域冲突觉察** |
| 批判性思维（Critical Thinking） | 教育哲学 | 批判性思维是内容分析，认知主权是**框架觉察** |
| 认知偏误矫正（Debiasing） | 行为经济学 | 偏误矫正是规则应用，认知主权是**元监控** |

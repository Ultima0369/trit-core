# 用户界面规格

**版本**: 0.1.0
**日期**: 2026-06-20
**状态**: 活跃
**分类**: 07_specs — 详细模块规格

---

## 一、设计原则

- **非侵入性**：界面不打扰用户，只在需要时提醒
- **可解释性**：每个输出都有"为什么"
- **可控性**：用户可配置显示内容和频率
- **美观性**：简洁、专业、不花哨

## 二、主要视图

### 2.1 注意力图谱（Attention Map）

**类型**：雷达图 / 力导向图
**数据**：各 Frame 的当前 Phase
**交互**：
- 点击节点 → 查看历史趋势
- 点击边 → 查看冲突详情
- 悬停 → 显示当前 MetaInterrupt

### 2.2 冲突面板（Conflict Dashboard）

**类型**：列表 + 详情
**数据**：当前所有 Hold 事件
**显示**：
- 时间
- 涉及 Frame
- 冲突类型
- 持续时间
- 建议行动

### 2.3 节奏报告（Rhythm Report）

**类型**：时间序列图 + 热力图
**数据**：基频、谐波、相位漂移
**周期**：周报告、月报告
**导出**：PDF / 图片

### 2.4 环境冲击预警（Shock Alert）

**类型**：卡片 + 进度条
**数据**：当前 ΔΦ、冲击等级、恢复进度
**状态**：
- 绿色：正常
- 黄色：中震（建议恢复）
- 红色：强震（强制 Hold）
- 灰色：毁灭级（建议专业支持）

### 2.5 决策审计（Decision Audit）

**类型**：时间线 + 详情弹窗
**数据**：每个决策的完整输入、运算、输出
**可追溯**：可查看任意历史决策的"为什么"

## 三、交互设计

### 3.1 首次启动（Onboarding）

1. 欢迎页：介绍 Aurora 的核心价值（不是"帮你做决定"，是"帮你看到决策被什么塑造"）
2. 数据源配置：选择要接入的数据源
3. 参考系设置：输入出生地、成长经历等（可选，可跳过）
4. 隐私确认：强调本地优先、数据不上云
5. 完成：进入主界面

### 3.2 日常交互

- 静默运行：后台采集和分析，不打扰
- 提醒时机：只在检测到重要变化时提醒
- 用户主动：用户可随时打开查看详细分析

## 四、技术实现

- 框架：Tauri（Rust 后端 + Web 前端）
- 前端：HTML/CSS/JS，可选轻量框架（如 Svelte 或纯 Web Components）
- 图表：D3.js 或 ECharts
- 样式：暗色主题为主，减少蓝光刺激

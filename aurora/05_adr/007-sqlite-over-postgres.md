# ADR-007：SQLite 作为本地数据库

**状态**: 已接受
**日期**: 2026-06-20
**分类**: 05_adr — 架构决策记录

---

## 背景

Aurora 需要本地数据存储。需要选择数据库。

## 决策

**采用 SQLite 作为本地数据库。**

## 考虑的选项

### 选项A：PostgreSQL

**优势**: 功能强大，并发性能好
**劣势**: 需要独立安装和配置，不符合本地优先
**结论**: 不可接受。太重量级，不适合本地优先。

### 选项B：LevelDB/RocksDB

**优势**: 键值存储，性能高，嵌入简单
**劣势**: 无 SQL 查询能力，关系数据建模困难
**结论**: 不可接受。关系数据需要 SQL。

### 选项C：SQLite ✅

**优势**: 零配置，单文件，嵌入简单，SQL 支持，跨平台，成熟稳定
**劣势**: 并发写入有限（但单用户场景足够）
**结论**: 可接受。完美匹配本地优先。

## 影响

- 数据存储：单文件 `aurora.db`
- 备份：文件复制即可
- 迁移：SQLite 版本控制 + 迁移脚本

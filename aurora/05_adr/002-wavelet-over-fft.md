# ADR-002：小波变换 vs 傅里叶变换 vs 阈值检测

**状态**：已接受  
**日期**：2026-06-20  
**分类**：05_adr — 架构决策记录

---

## 背景

Aurora 需要分析注意力的时间序列数据（通信频率、生理信号、行为模式）。需要选择一种数学工具来检测节奏变化、相位漂移、频谱重构。

## 决策

**采用小波变换（Wavelet Transform）作为主要分析工具，辅以傅里叶变换用于基频验证，阈值检测仅用于极端异常标记。**

## 考虑的选项

### 选项A：傅里叶变换（FFT）

**优势**：
- 算法成熟，计算快速（$O(N \log N)$）
- 频谱分析直观，有大量可视化工具
- 谐波检测简单（峰值频率的整数倍）

**劣势**：
- 假设信号平稳：无法处理非平稳信号（注意力数据 inherently 非平稳）
- 没有时间信息：知道"有哪些频率"，但不知道"这些频率什么时候出现"
- 频率分辨率与时间分辨率矛盾：长窗口频率准但时间模糊，短窗口时间准但频率模糊
- 无法检测瞬时变化：突变被平均到整个频谱中

**结论**：不可作为主分析工具。仅用于基频验证（已知周期下的精确频率测量）。

### 选项B：阈值检测（Rule-Based）

**优势**：
- 实现简单：if value > threshold then alert
- 计算极快：$O(1)$ 每次检测
- 可解释性强：规则直接对应业务逻辑
- 用户理解："他通话频率下降了50%"

**劣势**：
- 假设基线稳定：人的"正常"本身在变化（季节性、生命阶段、环境适应）
- 假阳性高：基线漂移被误判为异常
- 假阴性高：谐波结构改变但平均值不变
- 无法检测节奏变化：只能检测标量偏离，不能检测相位漂移
- 需要大量人工调参：每个用户、每个信号、每个时期都需要不同阈值

**结论**：不可作为主分析工具。仅用于极端异常标记（如通信频率突然降至零）。

### 选项C：小波变换（Wavelet Transform）✅

**优势**：
- 非平稳信号分析：频率可以随时间变化
- 时间-频率双重信息：知道"什么频率在什么时候出现"
- 多尺度分析：同时检测分钟级、小时级、日级、周级、月级模式
- 自适应窗口：高频用窄窗口，低频用宽窗口
- 相位信息：可以检测相位漂移（焦虑加速、抑郁减速）
- 瞬时变化检测：突变在小波尺度图上表现为垂直线
- 频谱重构检测：Kullback-Leibler 散度可以量化频谱变化

**劣势**：
- 计算更复杂：CWT 是 $O(N \log N)$，DWT 是 $O(N)$ 但实现更复杂
- 结果解释困难：小波尺度图不是直观可读的，需要训练
- 母小波选择：不同母小波产生不同结果，需要领域知识
- 边界效应：信号边界处的小波系数不可靠

**缓解劣势**：
- 计算：使用 Rust 的 `rustfft` 和自研 CWT，性能足够
- 解释：可视化层抽象（用户看到雷达图和趋势图，不看到小波系数）
- 母小波：默认 Morlet（平衡时间-频率分辨率），用户可选
- 边界效应：使用对称延拓或周期延拓，丢弃边界数据

**结论**：可接受。劣势可通过技术和 UI 设计缓解，优势不可妥协。

## 使用策略

| 分析任务 | 主要工具 | 辅助工具 |
|----------|----------|----------|
| 基频检测 | 小波（CWT） | 傅里叶（验证） |
| 谐波检测 | 小波（CWT） | 傅里叶（验证） |
| 相位漂移 | 小波（CWT） | — |
| 频谱重构 | 小波（CWT） | KL 散度（量化） |
| 跨信号同步 | 小波（CWT） | 同步指数（计算） |
| 极端异常 | 阈值检测 | 小波（验证） |
| 实时分析 | 小波（DWT，滑动窗口） | — |

## 影响

- **算法库**：需要集成小波变换库（Rust `rustfft` + 自研 CWT，或 Python `PyWavelets` 桥接）
- **数据管道**：原始数据 → 小波变换 → 特征提取 → Phase 归一化 → Trit-Core 输入
- **可视化**：需要小波尺度图（Scalogram）的渲染
- **计算资源**：小波变换比阈值检测消耗更多 CPU，需要优化热路径

## 验证

- 用已知节律的合成数据测试：小波能准确检测基频、谐波、相位漂移
- 用真实用户数据测试：小波检测到的变化与用户的自我报告一致
- 性能测试：在目标设备（普通笔记本电脑）上，实时分析延迟 < 1秒

# ADR-008：开源免费 vs 商业模式（重写）

**状态**：已重写（取代 2026-06-20 的"订阅制"决策）
**日期**：2026-06-30（重写）/ 2026-06-20（原决策）
**分类**：05_adr — 架构决策记录

---

## 背景

Aurora 需要选择分发与可持续模式。核心矛盾：如何让项目活下去，同时不背叛"防止注意力被收割、不剥夺用户选择"的价值主张。

原 ADR-008（2026-06-20）选择"订阅制"，并定义了个人版 ¥200 / 专业版 ¥2,000 / 团队版 ¥10,000+ / 企业版定制 四级付费层级，外加"决策审计服务 ¥5K-¥50K"。CTO_ROADMAP 据此在 M1-M4 的 Exit Criteria 中写入"至少 10 个付费用户""年收入 > 100 万人民币"等目标。

**该决策与项目根本动机冲突**：创始人在 2026-06-30 明确，本项目的立场是"放到 GitHub 开源，不要钱，不要名，不要一切注意力。已经有所困惑和洞察力的用户，是能够察觉本项目价值的。" 订阅制与付费层级隐含"获客—留存—变现"的注意力资本主义逻辑，正是本项目反对的东西。

## 决策

**全面改为开源免费（MIT），废止订阅制与所有付费层级。**

- 软件源码、文档、数据格式全部公开（MIT）。
- 不设订阅、不设付费层级、不设企业版定制、不设决策审计收费服务。
- 不主动推广、不投流、不 SEO 刷量、不追求 DAU / 留存 / 收入。
- 项目活下去的方式：创始人自持 + 自愿贡献者 + 可选的捐赠（捐赠不影响功能，不设捐赠门槛，不设"赞助商特权"）。

## 考虑的选项

### 选项 A：广告模式
**结论**：不可接受。广告就是注意力收割，与核心价值直接冲突。

### 选项 B：数据售卖
**结论**：不可接受。出卖用户数据 = 背叛认知主权（CHARTER 底线 1 不剥夺、底线 4 公开可审查）。

### 选项 C：订阅制（原 2026-06-20 决策）❌ 已废止
**为何废止**：订阅制本身不收割注意力，但"付费层级 + 获客 + 留存"的运营逻辑会**倒逼产品追求粘性与转化**，与"毕业即成功""不进化""不要注意力"三条根本立场冲突。一旦引入付费层级，反指标（用户依赖率应下降、打开时长不应追求）就成了与收入目标矛盾的摆设。

### 选项 D：开源免费 + 自我筛选 ✅
**优势**：
- 与"不要钱、不要名、不要注意力"完全一致。
- 目标用户是"已有困惑与洞察力的人"，这类人最反感被营销；不营销反而成为信任凭证（动机可审查）。
- 反指标不再与收入目标矛盾，可以严格执行。
- 用高门槛自我筛选用户，避免被未觉醒者稀释反馈质量。

**劣势**：
- 无商业收入，项目依赖创始人精力与自愿贡献者。
- 增长缓慢，无法靠资本加速。

**缓解劣势**：
- 严格限范围：不做 SaaS、不做多用户服务，单机工具维护成本低。
- 接受"项目可能很慢"作为正常状态——本项目不追求规模化，追求"对的人能用上"。
- 捐赠通道可选，但不与功能挂钩。

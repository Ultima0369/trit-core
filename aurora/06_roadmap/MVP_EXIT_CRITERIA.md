### M0 验收标准（Exit Criteria）

**目标**: 证明"小波 + 三值"概念可行，让有困惑与洞察力的人能自行找来并验证（开源免费、不推广，靠自我筛选）

**功能验收**

| 检查项 | 标准 | 验证方式 | 状态 |
|--------|------|----------|------|
| 邮件采集 | 能从至少一个邮件客户端读取元数据 | 手动测试 | ⏸ M1 — DataSource trait + JSON fallback 已完成，真实邮件 M1 |
| 小波分析 | 对合成数据能准确检测基频 | 自动化测试 | ✅ |
| Trit-Core 集成 | 跨 Frame 冲突时输出 Hold + MetaInterrupt | 自动化测试 | ✅ |
| HTML 输出 | 生成的 HTML 在浏览器中可正常显示 | 手动测试 | ✅ |
| CLI 可用 | 命令行参数正确解析，输出格式正确 | 自动化测试 | ✅ |
| 注意力调度 | 系统能输出 ShiftTo/HoldCurrent 提醒，用户选择被记录 | 自动化测试 | ✅ |
| 注意力图谱 | HTML 包含 ASI 仪表条 + 冲突面板 + 提醒历史 | 自动化测试 | ✅ |

**非功能验收**

| 检查项 | 标准 | 验证方式 | 状态 |
|--------|------|----------|------|
| 编译通过 | cargo build --release 成功 | CI | ✅ |
| 测试通过 | cargo test --workspace --all-features 全部通过（600+ tests） | CI | ✅ |
| 无 unsafe | `#![forbid(unsafe_code)]` + clippy clean | CI | ✅ |
| 伦理门禁 | cargo test ethics_ 10/10 全部通过 | CI | ✅ |
| 依赖审计 | cargo audit 0 漏洞 | CI | ✅ |
| 性能 | 日数据分析 < 1秒 | 手动计时 | ⬜ 待验证 |

**用户验收**

| 检查项 | 标准 | 验证方式 |
|--------|------|----------|
| 自我验证 | 作者能用 Aurora 检测到自己的通信节奏变化 | 亲历 |
| 早期反馈 | 至少 2 个外部用户试用并给出反馈 | 问卷/访谈 |

**风险与缓解**

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| 邮件客户端格式不兼容 | 高 | 中 | 支持多种客户端，提供手动导入 |
| 小波分析性能不足 | 中 | 中 | 优化热路径，必要时降级到 DWT |
| Trit-Core 扩展导致不兼容 | 低 | 高 | 严格保持向后兼容，扩展而非修改 |
